# 安全策略

## 报告漏洞

如果你发现安全漏洞，请**不要**提交公开 Issue。

请发送邮件至项目维护者，我们将尽快确认并修复。你将在漏洞修复后获得公开致谢（如果你希望）。

请在报告中包含以下信息：

- 漏洞的详细描述
- 复现步骤
- 受影响的版本
- 可能的修复建议（如有）

## 支持版本

| 版本 | 支持状态 |
|------|----------|
| `main` 分支 | 实验阶段，接收安全修复 |

> **注意**：本项目目前处于实验阶段，不保证向后兼容性。请勿在生产环境部署。

## 安全最佳实践

### 配置文件

- `config.toml` 包含 Access Token、数据库密码等敏感信息，已在 `.gitignore` 中排除
- 请勿将 `config.toml` 提交到版本控制系统
- 使用 `config.toml.example` 作为配置模板

### 插件安全

- **Python 插件**在独立子进程中运行，具有基本的进程隔离
- **Lua 插件**在嵌入式 VM 中运行，能够访问宿主进程的 API
- 仅加载来自可信来源的插件
- 插件的 `[dependencies]` 声明会触发 pip 安装，注意审查依赖包来源

### 网络通信

- 与 OneBot 实现的 WebSocket 连接使用明文通信（WASM 不支持 TLS）
- 建议在本机或内网环境中运行 OneBot 实现和 NeoBot
- Access Token 用于 WS 鉴权，请设置强随机 token

### 已知限制

- Lua 沙箱未严格限制系统调用
- Python 子进程可通过 pip 安装任意包
- 浏览器渲染功能会启动外部浏览器进程（Rod/Chrome）
